简单介绍

VuePress 是一个以 Markdown 为中心的静态网站生成器。你可以使用 Markdown 来书写内容（如文档、博客等），然后 VuePress 会帮助你生成一个静态网站来展示它们。

# 示例教程

这是一篇示例vuepress文档搭建，下面是几种方法供大家参考:

# 方法一

仔细参考官方文档 (opens new window)，按需引入即可

# 方法二

回到桌面 👉 双击打开Chrome图标 👉 在地址栏输入: https://www.baidu.com 👉 在搜索框中输入你的问题 👉 回车键

# 方法三（不去官网直接看这里）

# 像数 1, 2, 3 一样容易（官方是这么说的）
  1. 创建并进入一个新目录（建议在磁盘D/E上执行）
mkdir vuepress-starter && cd vuepress-starter
  1. 使用你喜欢的包管理器进行初始化
yarn init # npm init -y
  1. 将 VuePress 安装为本地依赖
yarn add -D vuepress # npm install -D vuepress

  1. 创建你的第一篇文档，VuePress 会以 docs 为文档根目录，所以这个 README.md 相当于主页：（README.md 也可在外面手动添加）

    mkdir docs && echo '# Hello VuePress' > docs/README.md

    编辑README.md

    ---
home: true
heroImage: /img/default.jpg
actionText: 博客笔记 →
heroText: 我的主页
actionLink: /home/
features:
- title: 简洁至上
  details: 使用最简单清晰的话语编写博客。
- title: 注重逻辑
  details: 知识点衔接紧密,注释明白,逻辑清楚。
- title: 兴趣驱动
  details: 丰富的代码案例,页面效果驱动学习。
  footer: 时间说真话 | © 耳畔所有
---

  2. 在package.json 中添加一些 scripts

{
  "scripts": {
    "dev": "vuepress dev docs",
    "build": "vuepress build docs"
  }
}
  1. 在本地启动服务器
yarn docs:dev # npm run dev

VuePress 会在 http://localhost:8080 (opens new window)启动一个热重载的开发服务器。 现在，已经有了一个简单可用的 VuePress 文档。接下来是 VuePress 中的基本配置。

# 基本配置

  1. 在你的文档目录下创建一个 .vuepress 目录，所有 VuePress 相关的文件都将会被放在这里。你的项目结构可能是这样：
.
├─ docs
│  ├─ README.md
│  └─ .vuepress
└─ package.json
  1. 在 .vuepress 目录下新建一个 config.js，它是VuePress 网站必要的配置文件，它导出的是一个 JavaScript 对象：
module.exports = {
  title: 'Hello VuePress',
  description: 'vuepress-starter blog',
  port: '8080'
}

终端执行命令: npm run dev，访问http://localhost:8080，你应该能看到一个页面，它包含一个页头，里面包含一个标题和一个搜索框，接下的操作是添加 导航栏。

# 更换主题

我们可以直接使用开源主题，例：这里使用的主题是 vuepress-theme-reco (opens new window)

  1. 安装 vuepress-theme-reco

    npm install vuepress-theme-reco --save-dev
# or
yarn add vuepress-theme-reco -D

  2. 引入主题，修改config.js：

    module.exports = {
  // themeConfig[...],
  theme: 'reco'
}

# 部署GitHub Pages

到这一步说明你的vuepress文档已经基本搭建好了✈️，接下来部署到免费的 Github Pages 上。首先在 Github 上新建一个仓库，例：这里取仓库名为：<USERNAME>.github.io，注意：<USERNAME>应该与你的GitHub用户名保持一致

  1. 修改config.js，添加一个base路径配置:
module.exports = {
  // 路径名为 "/<REPO>/" 这里需要跟仓库同名
  // base: '/<USERNAME>.github.io/',
  // 如果你打算发布到 https://<USERNAME>.github.io/，默认配值 "/"
  base: '/'
  //...
}
  1. 执行完以上操作后，最后你的config.js应该是这样的：
module.exports = {
  base: '/',
  title: 'Hello VuePress',
  description: 'vuepress-starter blog',
  port: '8080',
  markdown: {
    lineNumber: true
  },
  themeConfig: {
    nav: require("./nav.js"),
    sidebar: require("./sidebar"),
    collapsable: true,
    subSidebar: 'auto'
  },
}
  1. 在vuepress-starter目录下建立一个脚本文件：deploy.sh，注意修改一下对应的用户名和仓库名：
#!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run build

# 进入生成的文件夹
- cd docs/.vuepress/dist

git init
git add README.md # touch README.md
git commit -m 'first commit'

# 如果发布到 https://<USERNAME>.github.io
git push -f git@github.com:<USERNAME>/<USERNAME>.github.io.git master

cd -
  1. 修改package.json中的 scripts，添加:
"deploy": "bash deploy.sh"

到这里，你的目录结构应该是这样的：

.
├─ docs
|   └─...
├─ node_modules
|   └─...
├─ deploy.sh
├─ package.json
├─ package-lock.json
├─ README.md

最后在vuepress-starter 目录下，执行 npm run deploy，就会开始构建，然后提交到远程仓库的master分支，我们查看下对应仓库分支的代码。

⚠️⚠️⚠️注意：

这里运行deploy的方式为：通过右键Git Bash here，在此窗口下运行：npm run deploy

⚠️注意：如果通过上述方式运行出错了，建议手动上传远程仓库，详细操作参考：首页教程--置顶上传github文件

在仓库的 Settings -> Pages 中看到最后的地址，例：https://er-pan.github.io，时间说真话 (opens new window)，

⚠️注意：

按照上述步骤如果没有出现链接地址，此时未必是你的操作步骤上出错了，也有可能是网站的原因，这时需要等待20分钟过后再刷新页面就能看到链接了。
