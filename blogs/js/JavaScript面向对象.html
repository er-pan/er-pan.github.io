# javaScript面向对象基础

# 1. 面向对象简介

# 1.1 什么是对象

Everything is object（万物皆对象）。

  • 对象是现实世界中的一个实体：

    • 一本书、一辆汽车、一个人都可以是一个对象
    • 当实物被抽象成对象后，就可以模拟现实世界，针对对象进行编程。

  • 对象中只有属性和方法：

    • 对象的属性：对象的特征状态，是一个数据集。
    • 对象的方法：对象的行为，是一个功能集。

# 1.2 什么是面向对象

面向对象(Object Oriented,OO)，是一种编程模式和思想。它已经涉及到软件开发的各个方面。

  • 面向对象分析（OOA，Object Oriented Analysis）

  • 面向对象设计（OOD，Object Oriented Design）

  • 面向对象编程（OOP，Object Oriented Programming）

# 1.3 面向对象的实现方式

面向对象的两种实现方式：

  1. 基于类的（class）面向对象： java、C#、objectiveC、C++ ... ...

  2. 基于原型的 (prototype)面向对象： javaScript、Self、Lua ... ...

所以，Javascript 中没有类和接口，只有对象。

# 2. 创建对象

# 2.1 通过Object创建对象

Object是JavaScript的一个内置对象，可以通过Object创建对象：

示例： 
let zhangsan = new Object();
zhangsan.name = '张三';
zhangsan.age = 22;
zhangsan.eat = function(){
    console.log(this.name + '在吃饭');
}

console.log(zhangsan.name);     //张三
console.log(zhangsan.age);      //22
zhangsan.eat();                 //张三在吃饭

  • 使用 new Object() 创建对象。

  • 通过对象名，给对象添加属性或方法。

  • 在对象的方法中，使用 this 关键词调用自己的属性。

# 2.2 通过字面量法创建对象

{} 是字面量表示法的语法

示例： 
let zhangsan = {
    name:'张三',
    eat:function(){
        console.log(this.name + '在吃饭');
    }
}
//也可以通过对象名添加属性或方法
zhangsan.age = 22;

console.log(zhangsan.name);     //张三
console.log(zhangsan.age);      //22
zhangsan.eat();                 //张三在吃饭

只适合于创建单个对象的情况，所以也叫单体模式。这是一种快速构建一个对象的方式。

# 2.3 通过构造函数创建对象

2.3.1 创建对象

实际上，此种方式就是利用 function 创建的函数模拟一个类。这个函数叫做构造函数（Constructor）

示例： 
function Person(name){
    this.name = name;
    this.eat = function(){
        console.log(this.name + '在吃饭');
    }
}

let zhangsan = new Person('张三');
zhangsan.age = 22;
console.log(zhangsan.name);     //张三
console.log(zhangsan.age);      //22
zhangsan.eat();                 //张三在吃饭

  • 为了与普通函数有所区别，构造函数名首字母一律大写。

  • 使用 new 关键词，通过构造函数创建对象。

  • 实际上，Object就是一个通用的构造函数。

2.3.2 constructor属性

通过构造函数创建出来的对象，会自动包含一个constructor属性，此属性指向自身的构造函数。

示例： 
function Person(name){
    this.name = name;
    this.eat = function(){
        console.log(this.name + '在吃饭');
    }
}

let zhangsan = new Person('张三');
let lisi = new Person('李四');

console.log(zhangsan.constructor === Person);   //true
console.log(lisi.constructor === Person);       //true

2.3.3 instanceof运算符

instanceof 运算符: 验证原型对象与实例对象之间的关系。

示例： 
let zhangsan = new Person('张三');
let lisi = new Person('李四');

console.log(zhangsan instanceof Person); //true
console.log(lisi instanceof Person); //true

# 3. 原型对象prototype

在JavaScript中，prototype对象是实现面向对象的一个重要机制。

# 3.1 构造函数模式的问题

使用构造函数创建对象的方式存在一定的问题:

示例： 
function Person(name){
    this.name = name;
    this.eat = function(){
        console.log(this.name + '在吃饭');
    }
}

let zhangsan = new Person('张三');
let lisi = new Person('李四');

console.log(zhangsan.name===lisi.name);    //false
console.log(zhangsan.eat===lisi.eat);      //false

  • 代码中可以看出，从Person构造出的每一个实例对象，都拥有独立的属性和方法。

  • 每一个实例对象都拥有自己独立的name属性，这是合理的；

  • 但是，人人都应该有eat方法，也就是说：所有实例对象的eat方法都应该是同一个，而不应该是独有的。

也就是说：共通的属性或方法，应该是所有实例对象共用一个；只有非共通属性或方法，才应该是每个实例对象所独有。

# 3.2 原型对象prototype

javascript规定，每一个构造函数都有一个prototype属性，它指向一个对象。而这个对象的所有属性和方法，都会被构造函数的实例所继承。

这意味着，可以把那些共通的属性和方法，直接定义在prototype对象上。

示例： 
function Person(name){
    this.name = name;
}
Person.prototype.eat = function(){
    console.log(this.name + '在吃饭');
}

let zhangsan = new Person('张三');
let lisi = new Person('李四');

console.log(zhangsan.name===lisi.name);    //false
console.log(zhangsan.eat===lisi.eat);      //true
  • 从上面代码运行结果中可以看到：直接定义在构造函数中的属性或方法，是每个实例对象所独有的。而定义在prototype上的属性或方法，是所有实例对象所共有的。

# 3.3 prototype模式的验证

3.3.1 isPrototypeOf()

isPrototypeOf()方法用来判断，某个proptotype对象和某个实例之间的关系。

示例： 
let zhangsan = new Person('张三');
let lisi = new Person('李四');

console.log(Person.prototype.isPrototypeOf(zhangsan));  //true
console.log(Person.prototype.isPrototypeOf(lisi));      //true

3.3.2 hasOwnProperty()

每个实例对象都有一个hasOwnProperty()方法，用来判断某一个属性或方法到底是私有的（返回true），还是继承自prototype对象的（返回false）。

示例： 
function Person(name){
    this.name = name;
    this.study = function(){
        console.log(this.name + '在学习');
    }
}
Person.prototype.type = '人类';
Person.prototype.eat = function(){
    console.log(this.name + '在吃饭');
}

let zhangsan = new Person('张三');

console.log(zhangsan.hasOwnProperty("name")); // true
console.log(zhangsan.hasOwnProperty("type")); // false

console.log(zhangsan.hasOwnProperty("study")); // true
console.log(zhangsan.hasOwnProperty("eat"));   // false

3.3.3 in运算符

in运算符可以用来判断，某个实例是否含有某个属性或方法，不管是不是私有的

示例： 
function Person(name){
    this.name = name;
    this.study = function(){
        console.log(this.name + '在学习');
    }
}
Person.prototype.type = '人类';
Person.prototype.eat = function(){
    console.log(this.name + '在吃饭');
}

let zhangsan = new Person('张三');

console.log("name" in zhangsan); // true
console.log("type" in zhangsan); // true
console.log("study" in zhangsan); // true
console.log("eat" in zhangsan); // true

in运算符还可以用来遍历某个对象的所有属性

示例： 
let zhangsan = new Person('张三');

for(let prop in zhangsan){
    console.log(zhangsan[prop]);
}

# 4. 静态属性与方法

可以直接在构造函数上添加属性或方法。但是，这种形式添加的属性或方法是静态的。只能用构造函数本身来调用，而不能使用对象调用。

静态属性与方法多用与定义通用属性与方法。

示例： 
function MyMath(){}
MyMath.MALE = 1;
MyMath.FEMALE = 2;
MyMath.add = function(num1,num2){
    return num1+num2;
}

console.log(MyMath.add(10,20));
console.log(MyMath.MALE);

# 2. 面向对象编程之继承

# 1. 面向对象三大特性

面向对象有三大特性：

  • 封装：隐藏实现的细节。不考虑内部实现，只考虑功能的使用。

  • 继承：子类对象继承了父类对象的属性和方法，同时，子类对象也可以有自己的属性和方法。 继承实现了代码重用。

  • 多态：同一类型的对象可以呈现出多种形态。 js语言是弱数据类型，没有体现多态。当然，也可以说js语言中处处都是多态

# 2. 继承

# 2.1 call()与apply()方法

call()方法、apply()方法：用来代替另一个对象调用一个方法。就是更改对象的内部指针，即改变对象的this指向的内容。

实际上，就是可以让一个对象来执行另一个对象里的方法。

示例： 
function Class1(num1,num2){
    this.num1 = num1;
    this.num2 = num2;
    this.add = function(){
        return this.num1 + this.num2;
    }
}

function Class2(num1,num2){
    //Class1.call(this,num1,num2);
    Class1.apply(this, arguments);
}

let obj = new Class2(10, 20);
console.log(obj.add());

  • 在Class2构造函数中，使用Class1调用apply()方法，而且apply()方法的第一个参数是this（this指向的是Class2）。

  • 实际上，就是Class2来执行Class1中的属性或方法。

  • call可以传入多个参数；apply只能传入两个参数，所以apply的第二个参数是数组形式。

  • call()方法、apply()方法用来实现继承。

# 2.2. 实现继承

示例： 
function Person(name){
    this.name = name;
}
Person.prototype.eat = function(){
    console.log(this.name + '在吃饭');
}

现在想创建一个Student对象，希望能够继承Person的属性和方法。同时也想拥有自己的属性和方法。

示例： 
function Person(name,age){
    this.name = name;
    this.age = age;
}
Person.prototype.eat = function(){
    console.log(this.name + '在吃饭');
}

//子类构造函数
function Student(name,age,identity){
    Person.call(this,name,age);
    this.identity = identity;
}
//通过原型来继承父类的方法
Student.prototype = Person.prototype;
//加入子类自己的方法
Student.prototype.study = function(){
    console.log(this.name + '在学习');
}

//创建子类对象
let zhangsan = new Student("张三",25,"学生");
zhangsan.eat();
zhangsan.study();

//创建父类对象
let obj = new Person("张三",25);
obj.study();      //张三在学习

# 2.3 继承方式的优化

上面实现了继承。但是，以这种简单的方式实现继承，存在一个缺陷。这种方式导致Person和Student指向的是同一个原型，那么在子类中添加的方法，父类中同样可以使用。也就是子类影响了父类，这是不被允许的。

可以利用Object.careate()方法，先创建一个空对象，并且将这个空对象的原型指向Person.prototype，然后再赋值给子类。

Student.prototype = Object.create(Person.prototype);

# 2.4 原型链

什么是原型链：

1、每一个构造函数都有一个prototype属性, 指向他的原型对象

2、每一个实例化对象都有一个__proto__属性, 指向他所属类的原型对象

3、每一个原型对象都有一个constructor属性, 指向构造函数本身

# 2.5 方法重写

方法重写：子类继承了父类的方法，但如果认为该方法并不能满足子类的需求时，可以重新修改该方法的实现。新方法将覆盖旧方法。

示例： 
//父类的say方法：
Person.prototype.say = function(){
    console.log("hello " + this.name + "，你"+ this.age + "岁。");
}

//子类重写say方法
Student.prototype.say = function(){
    console.log("hello " + this.name + "，你"+ this.age + "岁。身份是：" + this.identity);
}

let stu = new Student('张博士',25,' 博士');
stu.say();

当调用Student对象的say方法时，会先查找子类对象上的say方法，如果找不到，才会去查找父类上的say方法

方法重写实例：

每个对象都有一个toString()方法。很明显，它是继承自Object的。toString()方法能够让我们以字符串的形式来打印一个对象。

示例： 
let zhangsan = new Person('张博士',25);
console.log(zhangsan.toString());

如果对继承的toString()方法不满意，那么我们可以重写它

Person.prototype.toString = function(){
    return "Person {" + "姓名:" + this.name + "，" + "年龄:" + this.age + "}";
}
let zhangsan = new Person('张博士',25);
console.log(zhangsan.toString());

# 3. 面向对象编程之封装

# 1. 封装数据

# 1.1 闭包

闭包（closure）是Javascript语言的一个难点，也是它的特色，很多高级应用都要依靠闭包实现。

在一个函数内部声明的变量，在函数外部是不可以访问的。

function fn(){
    let num = 1;
}
fn();
console.log(num);    //Uncaught ReferenceError: num is not defined

但是在某种需求情况下，需要在函数外部访问函数内部数据时，可以这样实现：

function fn(){
    let num = 1;
    function closure(){
        return num;
    }
    return closure;
}
let result = fn();         //获取了函数closure
console.log(result());     //调用函数closure，返回num内部变量。

  • 函数closure在函数fn的作用域内，所以，函数closure是可以访问函数fn中声明的变量。

  • 在函数closure中返回函数fn中声明的变量num，然后又将函数closure作为函数fn的返回值。

  • 这样，在函数fn的外部就可以访问到内部变量num。

什么是闭包：

  • 就是封闭了外部函数作用域中变量的 内部函数。

  • 但是，如果外部函数不返回这个内部函数，闭包的特性无法显现。如果外部函数返回这个内部函数，那么返回的内部函数就成了名副其实的闭包。

  • 此时，闭包封闭的外部变量就是自由变量，而由于该自由变量存在，外部函数即便返回，其占用的内存也得不到释放。

1.1.1 垃圾回收

变量不再使用后，如果不将变量所占用的内存回收，最终将导致内存耗尽，程序崩溃。这就是内存泄露问题。javaScript使用垃圾回收机制来处理内存泄露问题。

Javascript引擎基础GC方案是（simple GC）：mark and sweep（标记清除）:

  1. 遍历所有可访问的对象。

  2. 回收已不可访问的对象。

GC在回收内存时，首先会判断该对象是否被其它对象引用。在确定没有其它对象引用便释放该对象内存区域。当然，这一过程是按照固定的时间间隔周期性执行。

闭包中的垃圾回收：

示例: 
function fn(){
    let num = 1;
    function closure(){
        return num;
    }
    return closure;
}
/*
 * fn()执行后，如果没有闭包的存在，那么fn函数内的num变量将被GC回收。
 * 如果fn()中返回一个闭包，那么fn函数内的num变量仍然被引用，就不会被GC回收。
 */
let result = fn();
console.log(result());

# 1.2 使用闭包封装数据

使用闭包封装数据：

function AgeManage() {
    let age = 0;
    function setAge(param) {
        age = param;
    }
    function getAge() {
        return age;
    }
    return {
        'set': setAge,
        'get': getAge
    };
}

let age = AgeManage();
age.set(20);
console.log(age.get());

  • 首先，要将需要封装的数据，声明在函数中。

  • 在函数中声明两个赋值与取值的函数，并返回。这两个函数称为set方法与get方法。

  • 不能直接使用此数据，而是要通过set方法与get方法来操作此数据。

如果需要对数据进行验证，可以在set方法与get方法中完成：

function AgeManage() {
    let age = 0;
    function setAge(param) {
        //在set方法中对数据进行验证
        if(param>=0&&param<120){
            age = param;
        }else{
            age = 20;
        }
    }
    function getAge() {
        return age;
    }
    return {
        'set': setAge,
        'get': getAge
    };
}

let age = AgeManage();
age.set(200);
console.log(age.get());

# 2. 回调函数

回调函数：回调函数就是一个通过函数指针调用的函数。如果你把函数的指针（地址）作为参数传递给另一个函数，当这个指针被用为调用它所指向的函数时，我们就说这是回调函数。

# 2.1 实现回调函数

function fn1(callback) {
    console.log("做fn1函数本身的功能！");
    callback();
}
function endFn() {
    console.log("这是回调函数！");
}
fn1(endFn);

当fn1函数执行完毕后，调用回调函数，执行endFn()函数。

回调函数也可以带参数：

function fn1(callback) {
    console.log("做fn1函数本身的功能！");
    let num = 10;
    callback(num);
}
function endFn(param) {
    console.log("这是回调函数！并接收一个参数："+param);
}
fn1(endFn);
