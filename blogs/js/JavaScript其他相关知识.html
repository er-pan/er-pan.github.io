# Js 其它知识点

# 1. 异常处理

在ES3之前，js代码在执行过程中一旦出现错误，就会中断执行

console.log(b);
console.log('程序继续运行');

代码运行后，就会出现异常：

Uncaught ReferenceError: b is not defined

并且，程序会异常中断，不会继续执行后面的代码。

# 1.1 try...catch语句

try{
    //可能发生异常的代码
}catch(error){
    //处理异常的代码
}

  • 将有可能产生异常的代码放在 try 块中。这样，当发生异常时，异常对象就会被 try 块捕获，并交给 catch 块进行处理。

  • catch 块中可以进行异常处理。

  • error 参数引用了 Error 对象，此对象中包含了发生的错误信息。

try{
    console.log(b);
}catch(error){
    console.log(error);
}
console.log('程序继续运行');

上面代码中发生异常后进行了处理，所以程序可以继续运行了

# 1.2 finally语句

在实际开发中，某些功能完成后，是需要做一些最终处理工作的。比如：释放资源、回收内存等等。这些可以使用finally块来完成。

在try/catch块后加入finally块，可以确保无论是否发生异常，finally块中的代码总能被执行。

try{
    console.log(b);
}catch(error){
    console.log(error);
}finally{
    console.log('这里是finally块');
}
console.log('程序继续运行');

# 1.3 js中的错误类型

js代码可能会发生的错误有多种类型。当错误发生时，就会抛出相应类型的错误对象：

错误类型 说明
SyntaxError 解析代码时发生语法错误； 例如： console.log 'hello');
RefenrceError 引用一个不存在的变量； 例如： a.length;
RangeError 值超出有效范围； 例如： let arr = new Array(-1);
TypeError 变量或参数不是预期类型； 例如： let obj = {}; obj.a();
URIError URI相关函数的参数不正确； 例如： decodeURI('%1');
Error 以上错误类型都是Error的派生类。

# 1.4 throw主动抛出异常

在实际开发中，有时候我们在检测到一些不合理的情况发生时，也可以主动抛出错误。此时可以使用 throw 关键字主动抛出异常。

function converter(sex) {
    if(sex==0){
        return '男';
    }else if(sex==0){
        return '女';
    }else{
        throw new RangeError("性别只能为0或1！")
    }
}
converter(2);

代码运行后，会主动抛出一个错误：

Uncaught ReferenceError:性别只能为0或1！

# 2. 严格模式

严格模式是一种特殊的执行模式，它修复了部分语言上的不足，提供更强的错误检查，并增强了安全性。

在某个代码块中加入 “use strict”; 那么这个代码块中将执行严格模式。

"use strict";
x = 10;               //不允许使用未声明的变量。否则抛出异常。

"use strict";
function test(a,a){}   //不允许函数的参数重复定义。否则抛出异常。

# 3. 声明前置

使用 var 声明变量时会出现声明前置问题（也叫变量提升）

var num = 10;
function test(){
    console.log(num);
    var num = 20;
};
test();  //undefined

原因：javascript使用 var 声明变量具有hoisting机制（提升机制），JavaScript引擎在执行的时候，会把所有变量的声明都提升到当前作用域的最前面。但是，赋值却仍然在原来的位置。

上面代码就相当于：

var num = 10;
function test(){
    var num;
    console.log(num);
    num = 20;
};
test();  //undefined

# 4. 正则表达式

正则表达式：是用于描述一组字符串特征的模式，用来匹配特定的字符串。通过特殊字符+普通字符来进行模式描述，从而达到文本匹配目的工具。

# 4.1 创建正则表达式

创建正则表达式有两种方式：

  1. 使用RegExp对象创建正则表达式
//使用RegExp对象创建正则表达式
let reg = new RegExp('world'); 
let str = 'hello world';
console.log(reg.test(str));       //true
  1. perl风格创建正则表达式
//使用perl风格创建正则表达式
let reg = /world/; 
let str = 'hello world';
console.log(reg.test(str));       //true

正则表达式对象的 test 方法，会根据是否匹配返回一个布尔类型值。

# 4.2 边界控制

边界控制：匹配字符串开始和结束的位置。

● ^ 行首：匹配字符串开始位置(必须在方括号外面) --以什么开始。 ● $ 行尾：匹配字符串结束位置 --以什么结束。

let reg = /world$/; 
let str = 'hello world!';
console.log(reg.test(str));       //false

let reg = /world$/; 
let str = 'hello world';
console.log(reg.test(str));       //true

# 4.3 模式修饰符

模式修饰符 含义
i 忽略大小写
g 全局匹配。
表示该表达式将用来在输入字符串中查找所有可能的匹配,返回的结果可以是多个。如果不加/g最多只会匹配一个。
m 多行匹配。
匹配换行符两端的潜在匹配。影响正则中的^$符号。

忽略大小写的匹配：

let reg = /world/i; 
let str = 'hello WORld';
console.log(reg.test(str));       //true

全局匹配：

let reg = /world/ig; 
let str = 'hello WORld world';
console.log(str.match(reg));       //["WORld", "world"]

多行匹配（要和 ^ 或 $ 配合使用）：

let reg = /world$/; 
let str = 'hello world\n!';
console.log(str.match(reg));       //null

# 4.4 元字符

元字符就是拥有特定功能的特殊字符，大部分需要加反斜杠进行标识，以便于普通字符进行区别。

# 4.5 元字符范围

//查找移动手机号段(134至139号段):
let reg = /13[4-9]/g;
let str = "13856894256";
console.log(reg.test(str));

# 4.6 量词

//验证ATM机密码（必须为六位数字）：
let reg = /^\d{6}$/;
let str = "123456";
console.log(reg.test(str));

# 4.7 转义字符

转义字符：当需要匹配的字符串中包含. * + 等字符时，需要使用 \ 符号，将这些含有特殊意义的字符转换成普通字符。

//验证是否为合法的加法算式
let reg = /\d+\+\d+=\d+/;
let str = '75+38=113';
console.log(reg.test(str));

# 4.8 支持正则表达式的方法

总结一下支持正则表达式的 String 对象的方法：

let reg = /-/g; 
let str = "2020-12-24";
console.log(str.search(reg));        //4
console.log(str.match(reg));         //["-", "-"]
console.log(str.replace(reg,'/'));   //2020/12/24
console.log(str.split(reg));         //["2020", "12", "24"]

# 4.9 常用正则表达式

  1. 非空验证： /\S+/
  2. 是否为数字验证：/^-?\d+$/
  3. 邮箱地址： /^\w+(.\w+)*@\w+(.\w+)+$/i
  4. URL地址： /^(https?://)?(\w+.)+[a-zA-Z]+$/i
  5. 日期验证（不考虑闰年）：/[[1\]](#fn1)\d{3}-(0[1-9]|1[0-2])-(0[1-9]|[1-2][0-9]|3[0-1])$/
  6. 时间验证：/^(20|21|22|23|[0-1]\d):[0-5]\d:[0-5]\d$/
